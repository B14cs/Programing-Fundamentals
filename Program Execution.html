What is Program Execution?
Program execution is the process by which a computer runs your code. When you write a program and tell the computer to run it, the computer follows your instructions step by step, exactly as you wrote them.

üöÄ Key Concept

Programs execute sequentially - one instruction at a time, from top to bottom, unless you tell them to do otherwise with loops or conditions.

üîÄ Control Flow Changes Execution

Sometimes programs don't execute top to bottom. Control flow statements change the order:

üîÄ Conditions

If-else statements can skip certain lines of code based on conditions.

if (age >= 18):
    print("Adult") ‚Üê May skip this
else:
    print("Minor") ‚Üê Or skip this

üîÑ Loops

Loops repeat the same lines of code multiple times.

for i in range(3):
    print(i) ‚Üê Executes 3 times
‚Üê Then continues here

üìû Functions

Function calls jump to different parts of code and return.

def greet():
    print("Hi!") ‚Üê Jumps here

greet() ‚Üê Calls function
print("Done") ‚Üê Returns here

Understanding Function Execution Order

üí° Tip: The call stack keeps track of which function is currently running. When a function is called, it's added to the stack. When it returns, it's removed.

üîß Debugging Program Execution

When programs don't work as expected, you can:

  • Add print statements to see what values variables have
  • Use a debugger to step through code line by line
  • Check the console for error messages
  • Test small parts of your code separately

üí° Pro Tip

Understanding execution order is crucial for debugging. When your program doesn't work, trace through it step by step to see where it goes wrong!

üéì Key Takeaways

  • Programs execute instructions sequentially from top to bottom
  • Control flow statements can change execution order
  • Each instruction executes completely before moving to the next
  • Errors can stop execution or produce wrong results
  • Debugging helps you understand what your program is actually doing
  • The call stack tracks which function is currently running
  • When a function returns, its context is removed from the stack

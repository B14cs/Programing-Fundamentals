⚡ Program Execution

Understanding how programs run step by step

What is Program Execution?
Program execution is the process by which a computer runs your code. When you write a program and tell the computer to run it, the computer follows your instructions step by step, exactly as you wrote them.

🔄 The Execution Flow

1
📖 Load Program

Computer loads your program into memory and parses the code

2
🚀 Start Execution

Program starts at the first line of code and creates execution context

3
⚙️ Execute Instructions

Computer follows each instruction one by one from top to bottom

4
🔀 Handle Control Flow

Jump to different parts based on conditions, loops, or function calls

5
🏁 Program Ends

Execution stops when program reaches the end or encounters an error

🚀 Key Concept

Programs execute sequentially - one instruction at a time, from top to bottom, unless you tell them to do otherwise with loops or conditions.

How Code Gets Executed

💡 Tip: Execute code line by line, from top to bottom, unless directed otherwise by functions, loops, or conditionals.

🔀 Control Flow Changes Execution

Sometimes programs don't execute top to bottom. Control flow statements change the order:

🔀 Conditions

If-else statements can skip certain lines of code based on conditions.

if (age >= 18):
    print("Adult") ← May skip this
else:
    print("Minor") ← Or skip this

🔄 Loops

Loops repeat the same lines of code multiple times.

for i in range(3):
    print(i) ← Executes 3 times
← Then continues here

📞 Functions

Function calls jump to different parts of code and return.

def greet():
    print("Hi!") ← Jumps here

greet() ← Calls function
print("Done") ← Returns here

Understanding Function Execution Order

💡 Tip: The call stack keeps track of which function is currently running. When a function is called, it's added to the stack. When it returns, it's removed.

Program with Functions

function greet(name) {
  return "Hello, " + name;
}

function introduce() {
  let message = greet("Bob");
  console.log(message);
}

🐛 Common Execution Issues

🚫 Runtime Errors

Program stops executing when it encounters an error it can't handle.

x = 10 / 0 ← Program crashes here!
print("This never runs")

🔄 Infinite Loops

Program gets stuck repeating the same code forever.

while True:
    print("Forever") ← Never stops!

❌ Logic Errors

Program runs but produces wrong results due to incorrect logic.

# Trying to find average
sum = 10 + 20
average = sum / 3 ← Should be / 2!

🔧 Debugging Program Execution

When programs don't work as expected, you can:

  • Add print statements to see what values variables have
  • Use a debugger to step through code line by line
  • Check the console for error messages
  • Test small parts of your code separately

💡 Pro Tip

Understanding execution order is crucial for debugging. When your program doesn't work, trace through it step by step to see where it goes wrong!

🎓 Key Takeaways

  • Programs execute instructions sequentially from top to bottom
  • Control flow statements can change execution order
  • Each instruction executes completely before moving to the next
  • Errors can stop execution or produce wrong results
  • Debugging helps you understand what your program is actually doing
  • The call stack tracks which function is currently running
  • When a function returns, its context is removed from the stack
